# 工作空间主控制台（workspace-hub）布局优化与平台解耦设计文档

> 日期：2026-06-14
> 状态：设计已确认

---

## 1. 设计目标与原则

### 1.1 各司其职，超管解耦
* **彻底移除超管 Bento 卡片**：从控制台网格中完全移除原占 4 列的“超级管理员中枢”卡片，将主页面从管理员操作中解耦出来。
* **消息提醒徽标化**：在全局 `Header` 导航栏的用户头像下拉菜单（或已有的“管理后台”入口旁），增加实时红点/数字消息提醒（例如：`管理后台 (3)`）。当超级管理员登录时，通过并发拉取待审批的升级申请计数（`pendingApplicationsCount`）渲染提醒，点击后一键导流至专门的管理员后台。
* **清除多余按钮**：移除了主控台主体面包屑旁冗余的“注销/退出登录”物理按钮，统一通过全局 Header 的退出功能。

### 1.2 12 列 Bento Grid 黄金布局（桌面端 md:grid-cols-12）
* **第一行**：
  * **[卡片 1] 用户欢迎与个人名片 (8 列)**：展示用户名、根据会员级别特调的中文版勋章（如：`社区免费版` / `黄金会员版` / `钻石会员版`），以及当前活跃空间名片。
  * **[卡片 2] SVG 算力 Token 配额圆环 (4 列)**：展示算力已用百分比、具体数值及 VIP 升级导流。
* **第二行**：
  * **[卡片 3] 工作空间管理中枢 (8 列)**：融合了原先分离的个人主空间卡片和企业空间卡片。
  * **[卡片 4] 特权兑换与空间快速加入 (4 列)**：提供给普通用户输入协作邀请码或兑换特权的入口。
* **第三行**：
  * **[卡片 5] 舟坊高阶组件库 (8 列)**：展示部分高频使用的核心开发组件，支持 Hover 淡紫霓虹跃动，直接导流跳转至工坊。
  * **[卡片 6] 我的使用统计与 Top 3 看板 (4 列)**：展示累计调用等 4 个气泡，新增 Top 3 活跃组件调用频次看板，并自适应撑满与左侧组件库卡片等高，底部融合脉冲呼吸亮点的 SVG 趋势律动折线。

## 2. 详细页面结构与交互逻辑

### 2.1 会员等级徽标规范
* **等级文案汉化**：
  * `FREE` 渲染为 `社区免费版` (Badge 背景设为蓝紫色霓虹微光)。
  * `GOLD` 渲染为 `黄金会员版` (Badge 背景设为橙金色微光)。
  * `DIAMOND` 渲染为 `钻石会员版` (Badge 背景设为红粉渐变发光)。

### 2.2 工作空间管理中枢 (8 列)
* **Tab 切换分类**：
  * 顶部支持 Tab 切换：“个人空间”与“协作空间”。
* **空间额度强感知**：
  * Tab 栏右侧展示当前的**空间额度使用情况**（如 `个人空间 (1/1)`、`协作空间 (0/3)`）。
  * 额度限制根据用户的会员等级（FREE=0, GOLD=3, DIAMOND=10 等）进行动态计算。
* **按钮控制与拦截**：
  * 卡片右上角放置“新建个人空间”/“新建协作空间”按钮。
  * 若某类型的空间数量已达到额度限制，按钮在前端自动置灰，文案变更为“升级扩容 / 升级VIP解锁更多”。点击后阻止创建，并弹出升级 VIP 模态框，达成全流程拦截闭环。

### 2.3 使用统计与 Top 3 看板 (4 列)
* **消除留白**：通过 Flex 布局拉高卡片高度，使其与左侧的组件库卡片高度自动对齐，底色及边框完全对称。
* **活跃组件 Top 3 看板**：
  * 在卡片中部增加“高频调用组件 Top 3”实时数据行，展示用户最近使用最频繁的 3 个组件及其调用次数。
  * 鼠标悬浮行项时激活浅色渐变呼吸效果。
* **趋势图与亮点脉冲**：
  * 卡片最底部贴合渲染带有淡紫色渐变的微型 SVG 趋势折线。
  * 折线最高峰值点搭载一跳一缩的 `animate-ping` 脉冲呼吸动画发光原点，增强科技动感。

## 3. 后端数据模型与 API

### 3.1 聚合 API (`/api/user/workspace-hub/dashboard`)
在并行查询中，返回的数据结构进行如下增强：
* `pendingApplicationsCount`: 返回 PENDING 状态的超管申请计数（针对管理员角色）。
* `workspaceLimits`:
  * `personalCount`: 当前创建的个人空间数。
  * `personalLimit`: 个人空间创建上限（统一为 1）。
  * `enterpriseCount`: 当前创建的协作空间数。
  * `enterpriseLimit`: 协作空间创建上限（根据 membershipLevel 计算：FREE 为 0，GOLD 为 3，DIAMOND 为 10）。
* `topComponents`:
  * 返回最近被调用最多的 3 个组件的数据数组（包含 `componentId`, `name`, `callCount`）。

## 4. 最终落地与实现细节

本期 Bento 二期重构与解耦开发已于 2026-06-14 全部开发并验证完毕，详细落地细节如下：

### 4.1 后端 API 并行聚合扩展 (`/api/user/workspace-hub/dashboard`)
* **Top 3 活跃组件聚合**：在后端 `route.ts` 中利用 Prisma 的 `prisma.componentusage.groupBy` API，对 `userId` 下的所有组件调用进行组件 ID 分组并按调用次数降序排列，取前 3 名。
* **中文语义化映射**：后端声明了组件中文名称映射表（包含 `C01` ➔ `标书智能解析与售后打单`、`C02` ➔ `需求定义与产品设计` 等核心产品线），将原始 ID 在返回前端前转换成清晰直观的中文描述。
* **并发性能优化**：扩展原有的 `Promise.all` 基础并行查询，将 Top 3 聚合查询、个人/企业空间计数和平台待审批升级申请数合并为单次并发拉取，降低网络开销和数据库查询延迟。

### 4.2 全局 Header 待审批红点及待办气泡集成
* **超管身份拉取**：在全局组件 `Header.tsx` 中，用户登录并确认角色为 `ADMIN` 或 `SUPER_ADMIN` 后，触发对待处理审批计数接口的轮询与拉取。
* **头像右上角小红点**：当待审批升级申请数 `pendingCount > 0` 时，头像右上方将渲染一个发光的脉冲小红点（配合双层 `bg-red-505` 与 `animate-ping` 动感波纹）。
* **下拉菜单待办数量气泡**：在头像下拉菜单的“管理后台”行项，右侧醒目渲染红底白字的 `X 待办` 呼吸气泡，且整个入口点击即可直达审批后台，实现无缝消息提醒与快速处理导流。

### 4.3 工作空间 Tab 导航与额度拦截强感知
* **合并 Tab 展示**：将原本零散的个人工作空间与企业协作空间列表移出首屏，统一收拢到第二行 Card 3 中，顶部新增支持无缝切换的 Tab 分类（个人主空间 / 协作空间）。
* **额度动态计算**：Tab 栏右上方会实时展示当前的创建额度（如：`1 / 1` 或 `0 / 3`）。额度限制根据当前用户的会员等级在后端计算得出。
* **超限置灰强拦截**：在新建空间操作中嵌入强拦截，前端页面自动置灰按钮，文案变更为“升级VIP解锁更多”或“升级扩容”。用户点击置灰按钮将触发拦截并自动弹出 VIP 会员购买与升级模态框，保障全链路额度管控。

### 4.4 高度自适应与 Top 3 看板消除留白
* **组件等高对齐**：使用 Flex 容器的自动延伸布局使 Card 6 使用统计卡片在 md 屏以上完美和左侧的 8 列组件库卡片拉平对齐。
* **Top 3 排行看板填充**：在累计调用等 4 个指标气泡与底部折线之间，填充了最近调用 Top 3 组件列表，为每一项配合了精美的金/银/铜三色数字名次气泡和调用频次展示。
* **折线亮点呼吸效果**：保留了算力趋势折线图，同时使用 CSS `animate-ping` 亮点动画在折线的最高波峰顶点处渲染一个紫色发光的律动脉冲原点，为 Bento 注入了强烈的科技感和生命力。

### 4.5 留白缺陷修复与细粒度角色操作权限落地说明 (2026-06-14 补充)

为了进一步提升页面的科技美学与权限严密性，对控制台进行了以下高精度交互微调：
* **消除留白缺陷**：
  * **Card 1 优化**：在用户欢迎卡片下方引入用户邮箱和平台加入时间等元数据，紧凑排版，收缩由于与右侧配额仪表盘高度拉平而产生的冗余空白。
  * **Card 6 优化**：针对开发环境下可能出现的“暂无组件调用统计”占位进行精美化卡片重构，并微调底部 SVG 折线的高度占比与绝对定位，确保紧贴卡片底部，防止底部产生不自然的空白区。
* **重命名与汉化**：
  * 将前一版本的“协作空间”全部汉化重构为“企业空间”，确保与系统原先的专有名词体系完全咬合。
* **个人空间全态链交互找回**：
  * 重新激活并渲染个人空间下的“升级”操作按钮（当空间未升级且企业空间限额有余量时）。
  * 针对已被注销或已被平移升级的个人空间状态，在个人空间 Tab 下渲染“重新创建个人空间”功能入口（绑定原有 `handleRecreatePersonal` 逻辑），支持生命周期闭环。
* **企业空间多角色操作权限拦截**：
  * 引入 OWNER (所有者)、ADMIN (管理员)、MEMBER (普通协同成员) 三大角色的视觉 Badge（如 `所有者`、`管理员`、`协同成员` badge 气泡）。
  * 在操作按钮列实施与角色联动的严格前台置灰与 Tooltip 强拦截：
    * `进入空间`：对全员开放。
    * `扩容`：仅 OWNER 与 ADMIN 允许点击，MEMBER 状态下按钮置灰并提示“仅空间所有者或管理员可扩容此空间”。
    * `注销`：仅 OWNER 允许点击并调用 check-delete 和二次验证；ADMIN 与 MEMBER 状态下按钮置灰并提示“仅空间所有者可注销此空间”。
