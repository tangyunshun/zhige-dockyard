## 🧮 智能计算公式

### 计算方法

```typescript
function calculateDuration(message: string): number {
  let charCount = 0;
  
  for (const char of message) {
    // 中文字符（包括中文标点）权重 2
    if (/[\u4e00-\u9fa5]/.test(char)) {
      charCount += 2;
    }
    // 英文字母和数字权重 1
    else if (/[a-zA-Z0-9]/.test(char)) {
      charCount += 1;
    }
    // 标点符号和其他字符权重 0.5
    else {
      charCount += 0.5;
    }
  }
  
  // 基础时间 1500ms + 每字符 200ms
  const duration = 1500 + Math.round(charCount * 200);
  
  // 限制在 1500ms - 5000ms 范围内
  return Math.min(Math.max(duration, 1500), 5000);
}
```

### 计算参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 基础时间 | 1500ms | 保证用户至少能看到 1.5 秒 |
| 中文字符 | +200ms/字 | 中文阅读时间较长 |
| 英文字符 | +100ms/字 | 英文阅读时间较短 |
| 标点符号 | +50ms/个 | 标点阅读时间最短 |
| 最小值 | 1500ms | 防止显示时间过短 |
| 最大值 | 5000ms | 防止显示时间过长 |

  const message = data.message || "注册成功";
  
  // 显示提示（智能计算时间）
  toast.success(message);
  
  // 根据提示长度计算跳转时间
  const duration = 1500 + Math.round(message.length * 200);
  
  // 提示显示完后立即跳转（最长不超过 2.5 秒）

## 📈 时间范围控制

### 最小值：1500ms

**原因**:
- 人类阅读最短需要 1-1.5 秒
- 防止提示一闪而过
- 保证用户至少能看清提示

**适用场景**:
- 超短提示（如"成功"、"好的"）
- 用户非常熟悉的提示
- 重复性操作的提示

### 最大值：5000ms

**原因**:
- 避免用户等待焦虑
- 5 秒已经足够读完长提示
- 防止过长的提示阻塞操作

**适用场景**:
- 复杂的错误说明
- 包含解决方案的提示
- 重要的警告信息

### 跳转场景上限：2500ms

**原因**:
- 跳转场景需要更快的反馈
- 用户已经知道要跳转
- 避免打断操作流程

**适用场景**:
- 注册成功 → 登录
- 退出登录 → 首页
- 其他需要跳转的成功提示
