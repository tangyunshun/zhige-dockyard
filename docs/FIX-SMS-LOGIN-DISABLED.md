# 新注册账号登录被禁用问题修复报告

**修复时间**: 2026-05-05  
**问题等级**: 🔴 严重（新注册账号无法登录）

---

## 🐛 问题描述

**现象**: 新注册账号后，使用手机号验证码登录时显示"账号已被禁用"  
**影响**: 所有通过手机号验证码登录的新用户无法正常登录  
**用户反馈**: "为啥刚注册的账号还是去登录的时候就显示被禁用了呢"

---

## 🔍 问题根源

### 问题 API
**文件**: `src/app/api/auth/login-sms/route.ts`（手机号验证码登录）

### 问题代码（第 62-67 行）

```typescript
// ❌ 错误：直接拒绝所有非 active 状态的账号
if (user.status !== "active") {
  return NextResponse.json(
    { error: "账号已被禁用", status: "disabled" },
    { status: 403 },
  );
}
```

### 对比正确实现

**密码登录 API**（`src/app/api/auth/login/route.ts` 第 53-60 行）：
```typescript
// ✅ 正确：如果是 inactive 状态，自动激活
if (user.status === "inactive") {
  await prisma.user.update({
    where: { id: user.id },
    data: { status: "active" },
  });
  user.status = "active";
}
```

### 矛盾点

1. **注册时**: 用户状态设置为 `'inactive'`（未激活）
2. **密码登录**: 有自动激活逻辑 ✅
3. **短信登录**: **没有自动激活逻辑** ❌
4. **结果**: 新用户用手机号验证码登录时被拒绝

---

## ✅ 修复方案

### 修复逻辑
在短信验证码登录 API 中添加与密码登录相同的自动激活逻辑。

### 修复代码
**文件**: `src/app/api/auth/login-sms/route.ts`

**修改前**:
```typescript
// 删除验证码（验证成功后）
deleteSmsCode(phone);

// 检查账号状态
if (user.status !== "active") {
  return NextResponse.json(
    { error: "账号已被禁用", status: "disabled" },
    { status: 403 },
  );
}
```

**修改后**:
```typescript
// 删除验证码（验证成功后）
deleteSmsCode(phone);

// 如果是 inactive 状态，自动激活（新注册的用户）
if (user.status === "inactive") {
  await prisma.user.update({
    where: { id: user.id },
    data: { status: "active" },
  });
  user.status = "active";
}

// 检查账号状态（只允许 active 状态的账号登录）
if (user.status !== "active") {
  return NextResponse.json(
    { error: "账号已被禁用", status: "disabled" },
    { status: 403 },
  );
}
```

### 修复效果
1. ✅ 新用户首次使用手机号验证码登录时，自动激活账号
2. ✅ 只有真正被禁用（banned/deleted）的账号被拒绝登录
3. ✅ 登录成功后，账号状态永久变为 `active`
